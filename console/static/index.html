GEM² + Lobster Trap: Two-Layer AI Governance

TechEX Hackathon — Track 1: Agent Security & AI Governance (Powered by Veea)

Enterprise AI needs two trust layers: security and truth. Lobster Trap (L0/L3) blocks prompt-layer attacks at the wire in <15 ms. GEM² (L1/L2) audits reasoning-layer truth after the wire. Together: a four-gate governance pipeline that security and compliance teams can read.

The Two Trust Layers
🦞
Lobster Trap — Security (L0 + L3) "Is this safe?" — fast, deterministic policy enforcement. L0 scans the NL prompt at ingress; L3 scans the rendered output at egress. Regex DPI + 22 metadata fields + YAML policy. <15 ms, zero runtime cost.
🔬
GEM² Truth Filter — Compliance (L1 + L2) "Is this true?" — semantic + epistemic evaluation. L1 P-check validates input against type A & preconditions; L2 O-check validates output against type B & postconditions. Multi-provider consensus (Gemini, Claude, GPT-4o).
🤖
Contract-Executor (Vultr open-source LLM) Single-F worker between the gates. Vultr DeepSeek-V3.2 executes one contract's processing logic per stage. The contract holds; swap the model.
The Pipeline (per stage)
🛡️
L0 → L1 → F → L2 → L3 Five-gate sequence per node. L0 blocks malicious prompts; L1 validates input; F executes; L2 validates output; L3 blocks data exfil + credential leak in the rendered response. Every gate writes an audit trail.
🧩
Demo: Health-Insurance-Claim Pipeline 6 stages (intake → policy verification → eligibility → medical review → adjudication → disbursement). Each stage runs the full five-gate sequence. Judges inject adversarial inputs via the CE viewer and watch the gates fire.
📜
Auditable, regulator-ready Every verdict carries the rule that fired, the risk score, and the evidence retrieved. Regex policy is human-auditable YAML; LLM rationales are post-hoc evidence — not the source of truth.
